# Инструкция пользователя — собираем датасет стиля по шагам

> Понятное руководство для не-программистов.
> [English version](USER_GUIDE.md)

Этот инструмент превращает кучу страниц комикса в чистый, готовый к обучению
**датасет стиля** (папки картинок с подходящими текстовыми описаниями,
разложенные так, как ждёт тренер LoRA [kohya](https://github.com/bmaltais/kohya_ss)).
Он нарезает страницы на панели, стирает спич-баблы и звуки (SFX), выкидывает
дубли и слишком мелкие кропы и пишет подпись к каждой картинке.

Разбираться в коде не нужно. Просто следуйте шагам ниже.

---

## Что нужно

- **ПК с видеокартой NVIDIA** (GPU). Без неё тоже работает, но ИИ-шаги станут
  очень медленными.
- **Windows или Linux.** На Windows используйте **WSL** (Ubuntu) — бесплатный
  Linux внутри Windows. ([Как поставить WSL](https://learn.microsoft.com/windows/wsl/install) —
  одна команда и перезагрузка.)
- Около **15 ГБ свободного места** на диске (модели ИИ большие).
- **Интернет** на первый запуск (один раз скачать модели).

---

## Разовая установка

Делается **один раз**. Откройте терминал (на Windows — приложение **Ubuntu** из
WSL), скачайте проект и запустите скрипт установки:

```bash
# 1. Скачать проект (или кнопкой "Code → Download ZIP" на GitHub)
git clone https://github.com/hleserg/make-style-dataset.git
cd make-style-dataset

# 2. Установка «всё-в-одном» (ставит всё, в первый раз ~10–20 мин)
bash scripts/setup.sh
```

Эта одна команда ставит инструменты, скачивает зависимости, создаёт рабочие
папки и проверяет, что машина готова. В конце она печатает, что делать дальше.

> **Нет видеокарты NVIDIA?** Запустите `bash scripts/setup.sh --no-gpu`. ИИ-шаги
> пойдут на CPU (намного медленнее) — или прогоните их позже на машине с GPU.

Если захотите перепроверить машину, запустите:

```bash
uv run make-style-dataset doctor
```

Она печатает чек-лист — каждая строка должна быть `[ok]`.

## Сборка датасета

### Простой путь — приложение (рекомендуется)

После установки откройте приложение в браузере:

```bash
uv run make-style-dataset ui      # или:  make ui
```

Появится мастер из 3 шагов:

1. **Назовите стиль** — задайте триггер-слово (и число kohya-повторов).
2. **Добавьте страницы и соберите** — перетащите страницы комикса и нажмите
   **▶ Собрать датасет**. Лог прогресса идёт вживую. *Первый запуск качает
   несколько ГБ моделей — шаг может «висеть» на «running…» пару минут; это
   нормально, оно не зависло.*
3. **Заберите датасет** — посмотрите галерею результата и нажмите
   **Скачать .zip**. Отдельная вкладка **Manual review** показывает сложные
   страницы, отложенные для вас.

Это весь процесс, после установки терминал не нужен. Любите командную строку?
Шаги ниже делают ровно то же самое.

### Или из командной строки

#### Шаг 1 — добавьте страницы

Положите картинки страниц комикса (`.png`, `.jpg`, …) в папку
**`workspace/00_pages/`** (её создала установка). Одна картинка — одна страница;
чем больше страниц, тем лучше (цель — **от 20 страниц**, чтобы набрать достаточно
панелей).

#### Шаг 2 — запустите конвейер

```bash
uv run make-style-dataset run-all
```

**Первый** запуск качает модели ИИ (несколько ГБ) и идёт несколько минут.
**Это нормально — оно не зависло.** Следующие запуски гораздо быстрее.

В конце печатается сводка вида:

```
Pipeline summary:
  pages (00_pages)        24
  panels (01_panels)      96
  ...
  dataset (10_comicstyle) 83
  manual_review            11
```

#### Шаг 3 — заберите датасет

Готовый датасет лежит в **`workspace/05_dataset/10_comicstyle/`** — папка
картинок `.png`, у каждой рядом подпись `.txt`. Именно эту папку целиком вы
скармливаете тренеру LoRA kohya.

(`10` — число повторов, `comicstyle` — триггер-слово; как поменять — см.
*Настройки* ниже.)

---

## Две настройки, которые чаще всего меняют

Откройте файл **`.env`** (создан при установке) в любом текстовом редакторе и
поправьте при желании две строки:

```ini
APP_TRIGGER_TOKEN=comicstyle   # слово, которым стиль будет "вызываться" в промптах
APP_DATASET_REPEATS=10         # число повторов kohya; задаёт имя выходной папки
```

Сохраните и запустите конвейер снова. У всего остального — разумные значения по
умолчанию, их можно не трогать.

---

## Про папку `manual_review/`

**Полная автоматика тут нереальна — и это ожидаемо.** Примерно **1 страница из
7** сложная: сплэш на всю страницу, перекрывающиеся панели или бабл поперёк двух
панелей. Чтобы не гадать и не портить результат, инструмент откладывает такие в
**`workspace/manual_review/`**.

Откройте папку и посмотрите. Можно:

- **Проигнорировать** — если хороших панелей уже достаточно, вы закончили.
- **Поправить вручную несколько штук** — вырежьте хорошие панели сами и положите
  в `workspace/04_clean/`, затем снова запустите конвейер, чтобы их закаптионить.

**80–150 чистых панелей** — отличный датасет стиля. Спасать всё подряд обычно не
нужно.

## Если что-то не так

| Что видите | Что происходит / что делать |
|---|---|
| На первом запуске будто зависло | Идёт загрузка моделей ИИ (несколько ГБ). Подождите — не упало. |
| `doctor` пишет `[--] torch / CUDA` | GPU-стек не установлен или не видно видеокарты NVIDIA. Перезапустите `bash scripts/setup.sh` или используйте машину с GPU. |
| `doctor` пишет `[--] pages` | Папка `workspace/00_pages/` пуста — добавьте картинки страниц. |
| Очень медленно | Вы на CPU. ИИ-шагам нужна видеокарта NVIDIA. |
| `command not found: uv` | Закройте и снова откройте терминал, потом повторите (установка добавила `uv` в PATH). |
| Хочу начать заново | Удалите папку `workspace/` и выполните `uv run make-style-dataset init`. |

## Коротко

```bash
bash scripts/setup.sh                  # один раз
uv run make-style-dataset ui           # приложение: бросить страницы, Собрать, скачать .zip
```

…или всё из терминала:

```bash
#   положите страницы в workspace/00_pages/
uv run make-style-dataset run-all      # каждый раз
#   заберите workspace/05_dataset/<N>_<trigger>/
```

Технические детали каждой стадии — в [README-ru.md](../README-ru.md) и
[docs/architecture/SYSTEM.md](architecture/SYSTEM.md).
