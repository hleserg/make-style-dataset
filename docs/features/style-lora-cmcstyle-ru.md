# `cmcstyle` — рецепт стилевой LoRA на Flux.1-dev (HLE-803)

Готовый к запуску рецепт обучения и протокол оценки для стилевой LoRA `cmcstyle` —
результат задачи [HLE-803](https://linear.app/hleserg/issue/HLE-803) (родитель
HLE-802). Это **обучающая половина** (вопрос 3): подготовка датасета и капшнинг —
зона агента-датасета (см. [Предусловия от соседнего агента](#предусловия-от-соседнего-агента)).

Цель: LoRA, которая накладывает *манеру* комикса (пейнтерли-историческая BD —
приглушённая землистая палитра, акварельная заливка, тонкая чернильная линия) на
**произвольный контент**, НЕ запоминая конкретных персонажей, панелей и раскладок.

## Решение по тулингу: ai-toolkit (локально), не kohya

| | Локально (эта коробка) | Облако / GPU ≥ 24 ГБ |
|---|---|---|
| Тренер | **ai-toolkit** `/home/serg/ai-toolkit` v0.9.14 | kohya sd-scripts (стадия `train` в репо) |
| Почему | Доказано влезает и обучает Flux на 16 ГБ | Рецепт kohya применим там напрямую |

kohya sd-scripts **не обучает Flux на этой коробке 16 ГБ**: грузит весь fp8 DiT в
GPU и только потом оффлоадит swapped-блоки, поэтому `blocks_to_swap` не снижает
**пик** загрузки, а sub-fp8 кванта для трейна у этого kohya нет. К тому же 5070 Ti —
это живой 4K-дисплейный GPU (iGPU у i5-14600KF нет), ~1.5 ГБ постоянно держит
Windows `dwm.exe`; запас до OOM — пара сотен МБ и со стороны WSL не лечится.
ai-toolkit квантует на **CPU** и стримит блоки (`low_vram`) — пик загрузки GPU
остаётся низким; реальный fit-check выдал валидную 172 МБ Flux-LoRA без OOM.
Стадия `train` (kohya) в репо остаётся валидной для разового облачного прогона.

Сам рецепт гиперпараметров — **тул-агностичен**; различаются лишь написание флагов
и крит-для-влезания ручки (`qtype`/`low_vram`/оптимайзер).

## Конфиг

YAML для ai-toolkit. `<DATASET_DIR>` — папка `cmcstyle` от агента-датасета (см.
предусловия). Заземлён на проверенный fit-check (`/tmp/flux_fitcheck.yaml`) + синтез
research HLE-803.

```yaml
---
job: extension
config:
  name: "cmcstyle_flux"
  process:
    - type: 'sd_trainer'
      training_folder: "/home/serg/lora_output/aitk/cmcstyle_flux"  # вывод во владении serg
      device: cuda:0
      trigger_word: "cmcstyle"          # ai-toolkit добавит его в капшн, где его нет
      network:
        type: "lora"
        linear: 32                      # ранг — стилю нужна ёмкость (HLE-803: 16–32)
        linear_alpha: 16                # alpha/rank = 0.5 → консервативный эффективный LR
      save:
        dtype: float16
        save_every: 250                 # ПОшаговое сохранение → ~8 чекпоинтов на 2000 шагов
        max_step_saves_to_keep: 8
        push_to_hub: false
      datasets:
        - folder_path: "<DATASET_DIR>"  # датасет cmcstyle: картинки + контент-only .txt
          caption_ext: "txt"
          caption_dropout_rate: 0.05    # лёгкий дропаут капшна (анти-мемоизация, только ai-toolkit)
          shuffle_tokens: false         # порядок капшна не трогаем
          cache_latents_to_disk: true
          flip_x: true                  # зеркальная аугментация → стиль вместо композиции
          resolution: [ 512 ]           # авто аспект-бакеты; 768 — только после fit-check на 768
      train:
        batch_size: 1
        steps: 2000                     # 1500–2000; лучший чекпоинт обычно промежуточный
        gradient_accumulation_steps: 1
        train_unet: true
        train_text_encoder: false
        gradient_checkpointing: true
        noise_scheduler: "flowmatch"    # flux-shift объектив зашит внутри
        optimizer: "adafactor"          # почти нулевой стейт → влезает в 16 ГБ (доказано)
        lr: 1e-4
        dtype: bf16                     # bf16, НЕ fp16 (fp16 рискует пустой/NaN LoRA)
        disable_sampling: true          # ВЫКЛ: пресет 'generate' тянет ВЕСЬ трансформер
                                        # (+T5+VAE) на GPU разом (≠ стримящийся трейн-шаг) →
                                        # непроверенный риск OOM на 16 ГБ. Eval — пост-хок в
                                        # ComfyUI (см. Протокол оценки).
        ema_config:
          use_ema: false                # выкл — как в проверенном fit; опциональная ручка качества
      model:
        name_or_path: "black-forest-labs/FLUX.1-dev"   # gated diffusers-репо (кэш есть, токен есть)
        is_flux: true
        quantize: true
        qtype: qfloat8                  # НЕ qint4 (qint4+low_vram ломается: CUDA-only kernel при CPU-кванте)
        qtype_te: qfloat8
        low_vram: true                  # CPU-квант + стриминг блоков = влезание в 16 ГБ
      # ВАЖНО: блок инертен, пока disable_sampling=true. Промты ниже — это held-out
      # eval-набор, скорми их пост-хок ComfyUI-сетке. Сэмплинг во время обучения
      # включать только после короткого fit-check, что путь 'generate' влезает в 16 ГБ.
      sample:
        sampler: "flowmatch"
        sample_every: 250               # (только если сэмплинг включён)
        width: 512
        height: 512
        guidance_scale: 3.5             # сэмплим на реальном CFG ~3.5, НЕ на трейн-guidance
        sample_steps: 20
        seed: 42
        walk_seed: false                # фикс-сид → чекпоинты сравнимы
        neg: ""
        prompts:                        # held-out ЧУЖИЕ сюжеты + проверки утечки ([trigger]→cmcstyle)
          - "[trigger], a red sports car parked on a wet city street at night"
          - "[trigger], an astronaut riding a horse across a desert"
          - "[trigger], a bowl of steaming ramen on a wooden table"
          - "[trigger], a lighthouse on a rocky coast during a storm"
          - "[trigger], a fox sitting in a snowy pine forest"
          - "[trigger], a vintage steam locomotive at a station platform"
          - "[trigger], a portrait of an elderly fisherman, weathered face"   # проба утечки лица
          - "[trigger], a single robot standing alone in an empty field"      # проба баблов/рамок/SFX
meta:
  name: "cmcstyle_flux"
  version: '1.0'
```

## Запуск (только по сигналу пользователя — нужен ~весь VRAM 16 ГБ)

Запускать **онлайн** (НЕ ставить `HF_HUB_OFFLINE=1` — иначе ai-toolkit падает на
`model_info` вместо кэша):

```bash
cd /home/serg/ai-toolkit
venv/bin/python run.py /path/to/cmcstyle_flux.yaml
```

Скорость на коробке ≈ **7–9 с/it** под `low_vram` → прогон 2000 шагов ≈ **4–5 ч**.
GPU согласовать с соседним агентом (Flux хочет ~весь 16 ГБ). Чекпоинты — в
`training_folder/cmcstyle_flux/`, сетки-сэмплы рядом.

## Рационал гиперпараметров

| Параметр | Значение | Почему |
|---|---|---|
| ранг (`linear`) | **32** | Стилю нужно больше ёмкости, чем персонажной LoRA; дёшево против базы. До 16 — только если жмёт VRAM. |
| alpha (`linear_alpha`) | **16** | alpha/rank = 0.5 → эфф. LR ≈ 5e-5: консервативный, анти-оверфит режим. Поднять до 32, если стиль недопёкся. |
| `lr` | **1e-4** | Самый цитируемый LR для Flux-LoRA; он же в fit-check. Недопёк → 1.5–2e-4; перепёк/утечка → 5e-5. |
| `steps` | **2000** | Гайды по стилю — 1000–2500; лучший чекпоинт обычно промежуточный → выбирать по eval, не по последнему. |
| `resolution` | **[512]** | Стиль низкочастотен, хорош на 512, ~3× быстрее, меньше мемоизации панелей. Аспект-бакеты — авто. 768 — только если линия мягкая (свой fit-check). |
| `dtype` | **bf16** | Flux нативно bf16; fp16 рискует пустой/NaN LoRA. Ортогонально fp8-кванту весов. |
| `optimizer` | **adafactor** | Почти нулевой стейт → влезает в 16 ГБ (доказано). AdamW8bit — опция качества для облака/24 ГБ. |
| `noise_scheduler` | **flowmatch** | Включает flux-shift + guidance-distilled трейн внутри. Трейн-guidance флагом не передавать. |
| `qtype`/`qtype_te` | **qfloat8** | Изначальный блокер был не VRAM, а qtype: `qint4 + low_vram` падает (int4-kernel CUDA-only при CPU-кванте). qfloat8 кастится на CPU. |
| `low_vram` | **true** | CPU-квант + стриминг блоков держат пик загрузки GPU низким — обязательно на дисплейном GPU. |
| `flip_x` | **true** | Зеркало удваивает данные и толкает к стилю, а не к запомненной композиции; линия комикса ~зеркально-инвариантна. |
| `caption_dropout_rate` | **0.05** | Лёгкий дропаут капшна — доп. анти-мемоизация (только ai-toolkit). |
| `save_every` | **250 шагов** | Пошагово (не по эпохам) → стабильные ~8 чекпоинтов независимо от размера, чтобы поймать лучший промежуточный. |

## Предусловия от соседнего агента

Этим владеет **агент датасета/капшнинга**, но `cmcstyle` LoRA **не пройдёт DoD**,
если потребляемый датасет это не выполнит. Здесь только флаги, не реализация.

1. **Триггер = `cmcstyle`, не `comicstyle`.** Дефолт репо (`config.py`) и текущий
   датасет — `comicstyle`. Триггер вшит и в ведущий токен каждого капшна, и в имя
   папки, поэтому его надо выставить (`APP_TRIGGER_TOKEN=cmcstyle`) **до** стадии
   капшнинга — переименование папки потом даёт тихую пустышку (LoRA привяжется к
   тому слову, что ведёт капшны). `trigger_word` в ai-toolkit *добавляет* `cmcstyle`
   в капшны без него, так что чистая альтернатива — **контент-only капшны вообще
   без стилевого префикса**, а триггер впишет ai-toolkit.
2. **Капшны не должны называть стиль словами.** Текущий WD14-капшнер держит *все*
   general-теги, включая стиль/медиум (`monochrome`, `greyscale`, `comic`,
   `lineart`, `sketch`, `screentone`, `halftone`, `traditional media`,
   `speech bubble`). Назвать стиль словами — убить премис Style-Locker: стиль должен
   привязаться к НЕописанному триггеру, а не к частым словам, что встречаются в
   произвольных промтах. Лечение: блоклист стиль/медиум-тегов + поднять порог
   0.35 → ~0.65, **или** контент-only NL-капшнер. Самый влиятельный фикс капшнов,
   применим к любому капшнеру.
3. **Описывать повторяющихся персонажей (вариативно), чтобы убрать их из триггера.**
   Принцип «опиши, чтобы исключить»: описанное привязывается к (меняющемуся)
   тексту; неописанное — к триггеру. Поэтому явное и *вариативное* описание
   повторяющихся людей («молодой мужчина с косичками», «римский солдат в доспехе») —
   именно то, что мешает стилю их запомнить. Не давать персонажу фикс-имя/токен.
4. **Курировать на разнообразие (анти-мемоизация ручка №1).** Цель — 20–40
   *максимально разных* кропов (разные персонажи, сцены, ракурсы, палитры). Ни один
   повторяющийся субъект > ~20–25 % набора. pHash-дедуп убирает почти-идентичные
   кадры, но не «тот же персонаж, другая поза» — нужен дополнительный per-subject кап.
5. **Не пускать кропы с протёкшим гаттером.** Слайсер S1 затягивает соседний контент
   на диагональных/нерегулярных гаттерах (видны линии-разделители / две суб-панели в
   одном кропе). Это вносит панельную структуру в обучающие кропы — кропы с низким
   fill-ratio гнать в `manual_review`.

## Протокол оценки

**Основное: пост-хок в ComfyUI** по сохранённым чекпоинтам (инференс Flux.1-dev на
этой коробке уже доказан). Сэмплинг во время обучения оставлен **выкл**: его пресет
`generate` тянет весь трансформер (+T5+VAE) на GPU разом — в отличие от стримящегося
трейн-шага — непроверенный риск OOM на этом дисплейном GPU 16 ГБ. Промты из
`sample.prompts` служат held-out eval-набором для сетки. (Чтобы включить превью во
время обучения — сначала короткий fit-check сэмплинга, см. Откаты.)

- **Held-out / чужие промты** — сюжеты, отсутствующие в обучающих кропах. PASS =
  стиль комикса (линия, тени, палитра) переносится, при этом **ни один** конкретный
  обучающий персонаж/костюм/лицо/сцена не возвращается.
- **Проба комикс-утечки** (обязательна здесь) — одно-субъектные промты НЕ должны
  возвращать панельные рамки, гаттеры, мульти-панельные сетки, пустые баблы или
  SFX/звукоподражание. Непрошеные баблы/рамки — сильнейший признак, что триггер
  вобрал раскладку вместо стиля.
- **Финальная сетка фикс-сид «вес × промт»** (для репорта) — X = вес LoRA
  `{0.0, 0.5, 0.7, 0.85, 1.0}`, Y = промты. Колонка **вес 0.0 (база) обязательна**
  как бэйзлайн, доказывающий, что стиль *добавлен*. Сэмплить на CFG ~3.5. Собрать в
  ComfyUI (blueprint Flux.1-dev + XY по силе LoRA) или SwarmUI XYZ.
- **Выбор чекпоинта**: **самый ранний**, где стиль уже закрепился, но следование
  промту и разнообразие сюжетов ещё живы (стилевые LoRA пикуют рано; последний
  чекпоинт обычно *не* лучший).
- **Вес на инференсе**: наибольший вес, ещё проходящий чужие + утечка-пробы —
  ожидаемо ~0.7–0.85.

## Маппинг на DoD (HLE-803)

| Пункт DoD | Как рецепт его закрывает |
|---|---|
| Накладывает стиль на произвольный контент | Сетка чужих промтов; стиль привязан к неописанному триггеру. |
| Триггер работает | `cmcstyle` ведёт/вписан в каждый капшн; проверяется в каждом sample-промте. |
| **Не** протаскивает персонажей/панели | Кураторское разнообразие + «опиши-чтобы-исключить» + flip + утечка-пробы + ранний чекпоинт. |
| Настройки задокументированы | Этот файл (конфиг + рационал + зафиксированные значения). |
| Формат датасета = контракт HLE-802 | `<dataset>/{img,txt}`, контент-only капшны, мульти-аспект бакеты, триггер `cmcstyle`. |

## Откаты

- **Пустая LoRA** (крошечный файл / нулевые тензоры) → баг точности/кванта: проверь
  `dtype: bf16` и `qtype: qfloat8` (не qint4).
- **Недопёк** (веса есть, стиль слабый) → подними `lr` до 1.5–2e-4, или
  `linear_alpha` 16 → 32, или возьми более поздний чекпоинт.
- **Перепёк / утечка контента** → опусти `lr` до 5e-5, понизь вес на инференсе или
  возьми более ранний чекпоинт.
- **Линия мягкая** → добавь `768` в `resolution` (сначала перепроверь влезание).
- **Сэмплинг во время обучения ловит OOM (~шаг 250)** → ожидаемо, если переключить
  `disable_sampling` в `false`: пресет `generate` материализует весь трансформер на
  GPU. Держи `disable_sampling: true` и делай eval пост-хок в ComfyUI; включать
  сэмплинг только после короткого fit-check (~260 шагов, чтобы пересечь границу
  `sample_every`), доказавшего влезание — этот fit-check уже «трейн», запускать в
  открытом пользователем GPU-окне.

> Эвристические числа (вес инференса 0.7–0.85, per-subject кап 20–25 %, дропаут
> 0.05) — стартовые точки для калибровки по eval-сетке, не жёсткие константы.
